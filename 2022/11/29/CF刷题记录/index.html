Social Network

Problem - D - Codeforces

有 n 个人，m 对关系，i 从 1 到 m ，每次可以指定两个人联系起来，但最终的结果必须满足前 i 对关系。求对于每个 i ，这个人可以联系起来最多数量的人数。

很明显的并查集求最大连通块，对于一个 i ，如果第 i 对关系在此次指定联系之前已经满足，则这次可以指定较大的两个连通块相联系。每联系一次就重新算一次所有连通块的大小。时间复杂度 O(n^2)。

Robot on the Board 1

Problem - E - Codeforces

一个机器人在 n*m 的二维平面上行走，给出了机器人接下来的行走方向，求机器人的初始位置在哪里时，走的步数最大（掉出二维平面之前）。

直接模拟机器人的行走方向，维护行走区块的最大上下左右边界。

Moderate Modular Mode

Problem - D2 - Codeforces

给出两个偶数 x , y ,找出一个正整数 n 使得 n%x = y%n成立

分情况讨论，x = y时明显 n = x，

x > y 时，n = x+y；

x < y 时，若 n > y ，n%x < x，y%n = y不成立

若n < x， n%x = n, y&n < n不成立

故 n 一定在 x 到 y之间。通过画图可以看出，把y-x+1以长度为x均分，n就在最后一段的中间。

Gardener and Tree

Problem - E - Codeforces

有一颗无根树， 结点编号从 1 到 n ，你可以进行 k 次操作，每次操作可以剪掉当前的所有叶子结点。求 k 次操作后这颗树还剩下多少结点。

每次操作相当于把这棵树剥下来一层，那么可以用bfs去写，初始化把所有入度为 1 的点删除，并push进队列，同时记录这是第1次操作。对于队列中的一个结点，让与它相连的所有结点的入度 -1，当与它相连的结点的入度为 1 时，删除它，操作数 +1，然后push进队列。因为入队条件是入度为1，所以特判一下只有一个结点的情况。

Vupsen, Pupsen and 0

Problem - D - Codeforces

现在有一个数组 a，每个元素在 -1e4 ~ 1e4之间，且不含0，求构造另一个数组 b ，让ai * bi 的和为0。

题目要求 b 数组的绝对值之和不能超过1e9，那么极限数据来看，是不能对 a 数组进行任何加乘操作的，只能由原本的数来改。对于任意两个数a,b，a b + (-a b) = 0，因此对于n为偶数时，可以直接两两相互构造。对于n为奇数时，考虑前三个数a,b,c，设ax + by +cz = 0，当x = (b+c) && b+c!=0 && y = -a && c = -a时成立，然后再两两构造即可。

Carrying Conundrum

Problem - C - Codeforces

现在定义一个另类加法，在原加法的基础上，定义进位：向前一位的前一位进一。现在给你另类加法的结果n，求有几种（a , b）对，使得加法结果为n。

观察加法列式，发现可以分成奇偶，奇数位进位到下一奇数位，偶数位进位到下一偶数位。两种情况相乘，再减去全0的情况即为答案。

Seating Arrangements (hard version)

Problem - D2 - Codeforces

在一个电影院里排队看电影，座位是一个n*m的二维平面，从左到右依次有一个参数pi，每一个人也有一个参数ai，ai越低就需要安排到较低的pi的位置，也就是说，任意ai，aj，若ai<aj，那么pi<pj。这些人需要按顺序坐到自己应该坐的位置，他们会先来到对应的一排，再从左到右进入到自己的位置，如果一个人在前往自己的位置的过程中遇到了一个已经坐好人的位置，就会让他尴尬，怎么样安排座位才能让整体尴尬次数最少。

因为对于任意ai，aj，若ai<aj，那么pi<pj。所以最终的座位安排一定是确定的，先按照大小从小到大排序，大小相同的按照编号从小到大排序，这样就可以把这些人分成m组，每一组具体座位再单独安排，对于某一组，较小ai的一定坐在靠左位置，若ai相同，编号大的先进去坐相对靠右的位置。每安排好一个位置，就遍历一遍之前安排好的，计算这个人会经过几次已经有人的座位。时间复杂度O(n^3)。

Say No to Palindromes

Problem - D - Codeforces

给你一个只由[a,b,c]组成的字符串 s ，m个询问，每个操作给出 l，r，要求子串 [sl……sr] 不能存在长度大于 2 的回文串，你可以选择子串中的一个字符改变成[a,b,c]中的一个。问每次询问最少要操作几次？每次询问独立。

因为字符串只由三个不同的字符，所以当字符串较长时，可以明显的观察到不存在回文串的情况是三个不同的字符循环排列，排列只有6种情况“abc”，“acb”，“bac”，“bca”，“cab”，“cba”，分别用这些循环节去遍历字符串，前缀和统计出每种循环节需要让原串改变多少字符，每次操作从6种情况里取最小值。

Secret Santa

Problem - D - Codeforces

有 n 个人相互给对方送礼物，每个人都有个期望对象 ai ，他们希望自己的礼物送给 ai ，但是一个人只能收到一个礼物，且不能把礼物送给自己，求怎样安排才能让更多的人把礼物送给期望的人。

每个人只能收到一个礼物，先把可以送给期望 ai 的礼物送出去，这就已经是最多的满足期望数了，剩下的人先按照顺序送给还没有拿到礼物的人，这样就可能出现一个问题：有人会自己给自己送礼物。那就把这个人 i 的礼物送给他期望的人 ai ，然后原本要送给 ai 礼物的人送给 i，这样形成闭环，且满足期望数不变。

Erase and Extend (Hard Version)

给你一个字符串 s ，你可以进行两种操作任意次：删除 s 末尾的字符， 或者让 s = s + s，求字典序最小的字符串s变成长度 k 的新串。

肯定是先进行若干次 1 操作，然后再进行 2 操作直到长度满足 k （多余的删掉）。

因为要让串翻倍，所以可以看第一个字符p，遍历这个字符串，如果 p > i 说明若字符串从 i-1 开始循环一定比从 i 开始循环的字典序要小，若 p < i ，说明字符串从 i 开始循环 一定比从 i-1 开始循环的字典序要小，若 p = i，就让p的下一个字符去比较。

p > i 的时候就跳出遍历，答案串从循环到 i-1的位置就行了，若p不为原串的第一个字符，说明有 p -1个前缀字符算上去了，要让 i - (p-1) 。

PriceFixed

Problem - D - Codeforces

n 件物品，每件物品需要买 ai 个，每次购买花费 2 硬币，如果此前已经购买了 bi 个物品，此次购买花费 1 硬币，问怎样买能让花费最小？

贪心一下，按照 bi 从小到大排序，如果已经购买的数量 cnt < bi 就先买后面的，凑数量，这样再购买前面的就可以少花钱。

Sequence Pair Weight

Problem - C - Codeforces

定义一个数组的权重为 (i,j) 对的数量，使得 ai = aj ，现在给你一个数组，求它所有子数组的权重和。

因为是相同数字才能计算贡献，所以先把所有相同值不同位置统计出来，对于相同数字的任意两个位置 i,j ，选择这两个位置后左右边界可以扩大到最左或者最右，那么这两个位置对整个数组的贡献就是 i(n-j+1) ，然后再看相同数字的第三个位置 k ，它可以与前面第一个或者第二个组合，贡献是 i (n-k+1) + j (n-k+1) ，即 (i + j) (n-k+1) ，然后就总结出来了一个线性统计相同数字不同位置对整个数组的贡献方法。

Potions (Hard Version)

Problem - C2 - Codeforces

给你 n 个数，你要从左到右依次选取这些数，对于一个数，你可以选择选还是不选，在这个过程中你要保证你所选择的数字总和是非负数，问最多可以选择多少个数字。

首先对于每一个非负数肯定要全选，然后贪心策略是尽可能拿较大的负数，这样可以保证往后多拿点，但是这个负数能不能拿取决于我已经拿的数的总和，不能因为拿了这个数让总和变成负数了，所以要在线操作，边拿边判断，对于一个负数 ai ，如果我现在的总和 sum + ai >0 ，我可以先拿掉这个负数，然后再往后看，如果遇到了 sum + ai < 0，就从我前面拿过的数中比较最大的数，如果这个数更小点，显然是选这个更合理，然后把贡献重新算一下。

Permutation by Sum

GCD Queries

Problem - D - Codeforces

这是一个交互题，有一个长度为 n 的隐形数组，它是一个由 0~n-1 的数字组成的序列，你可以对数组进行不超过 2n 次查询，每次查询可以选择两个下标 i,j，系统会返回 gcd(i,j) 的值。请你在查询完成后返回答案：一对下标i,j，其中 。

交互题可以从查询次数入手，对序列查询不超过 2n 次，每次返回的是 gcd ，那么很大的可能是每次查询两对下标，根据返回值去判断排除掉一个绝对不为 0 的位置。

初始化答案 x=1,y=2 ，遍历 i>2 的所有下标，每次询问 (x,i) (y,i)，返回 X = gcd(x,i) , Y = gcd(y,i) 。

那么我们要从这三个数中排除掉一个无论如何都不可能为 0 的数。因为数组的隐形的，所以返回值只有比大小存在意义，那么就有三种情况：

如果 X = Y，i 绝对不可能是0，因为隐形数组是序列，每个值都不同，如果 i=0 则 gcd(x,i) != gcd(y,i)。

如果 X > Y，i 可以是 0，y 绝对不可能是 0，因为 gcd 要么等于两者中最小值，要么比最小值还小，如果 y = 0，一定有 Y = gcd(0,i) = i >= gcd(x,i) = X，即 Y > X 与返回结果相反。

如果 X < Y，x 绝对不可能是 0，原理同上。

Coloring

Problem - B - Codeforces

有 n 个方块，m 种颜色，要求对方块涂色的同时任意连续的 k 个方块颜色两两不同，要求每种颜色使用 ai 次，所有颜色使用完刚好吧 n 个方块涂满。问所给的 n m k 能否满足上述要求。

很容易可以想到吧 n 个方块分成 k 份，让每一份中的方块颜色都两两不同，同时每种颜色的使用次数不超过 k 份，那么只要使用次数最多的能涂上去，后面的也能涂上去。

但是这里有一个易错点：n/k 应该是向上取整，就是如果最后一份不满足 k 个方块要统计一下恰好能填满 n/k 上取整个区域的颜色种数，这个颜色种数应该满足小于等于 n%k 。因为虽然最后一份不足k个方块，但它已经在最后了，后面没有多余的方块，即使不足 k 个方块也能满足连续k个方块俩两不同。

Same Count One

Problem - D - Codeforces

给你 n 个长度为 m 的二进制字符数组，你可以进行操作任意次：选择两个数组和一个位置，$swap(a{ik},a{j,k})$。使得每个数组中 1 的个数相同。

首先，如果全部数组中 1 的个数不能整除 n 那么就是无解，否则就是吧 1 多的放到 1 少的数组里。

观察这些二进制字符数组发现，对于一个 1 数量多的数组 a ，和一个 1 数量少的数组 b，一定可以找到一些位置 i 使得 ai 一定是多余的 1 ，bi 一定是多余的 0 ，这样就可以直接交换他们，总的交换次数一定是最少的。

那么我们就可以一列一列的看，每列创建两个queue，对于这一列 i，如果数组 a 中 1 的数量多，且刚好 ai = 1，那么这一行数组 a 就可以放入待交换队列1，如果数组 b 中 0 的数量多，且刚好 ai = 0，那么这一行数组 b 就可以放入待交换队列2。一整列遍历完后将交换队列中的待交换行两两匹配，如果匹配不满就把多余的舍弃，待到后面列去重新寻找匹配。

Balance the Bits

Problem - 1503A - Codeforces

给你一个长度为 n 的01字符串，构造两个合法括号序列 a b，要求若 si = 1，ai = bi，若 si = 0，ai != bi。

对于一个合法的括号序列，首尾肯定是一左一右括号，是确定的，所以若s1=0||sn=0则无解。

可以简单的想到先去构造确定的 1 位置，由合法的括号序列增加未知括号序列使其仍然合法，那么 0 的位置构造后除了前后两个肯定也是合法的，所以 1 和 0 的数量就必须是偶数的。构造1的合法可以直接对半分开，前面左括号后面右括号，构造0的合法直接一左一右来回换。

k-LCM (hard version)

Problem - C2 - Codeforces

构造 k 个数，使得 k 个数的和为 n，且他们的 LCM <= n/2.

抓住 LCM <= n/2 这一点，如果 k = 3，且 n 是奇数的情况可以直接构造出 n/2,n/2,1 这三个数，如果 n 是偶数，就看 n/2，如果 n/2 是奇数就构造 n/2-1,n/2-1,2 ，如果 n/2 是偶数，就构造 n/2,n/2/2,n/2/2 。

这是 k = 3 的情况，如果 k>3 就把大于 3 的部分都构造成1。

Square-Free Division (easy version)

Problem - E1 - Codeforces

给你一个长度为 n 的数组，你要把他分成几个连续的子数组，使得任意一个子数组中任意两个数的乘积不会是个平方数。求最少能分成几个子数组。

对平方数分解质因子会发现它每一个质因子的幂次都是偶数，那么我们在遍历数组的时候如果遇到当前数所有的的奇次幂质因子加上前面某个数所有的的奇次幂质因子刚好变成了偶次幂，就说明他们两个的乘积是个平方数，所以对于每一个数我们分解质因子，只记录它奇次幂质因子的种类，乘积起来做个标记，如果后面遇到了就说明需要切一刀。

Guessing the Greatest (hard version)

Problem - C2 - Codeforces

给你一个隐藏的数组，每次询问一个区间 l,r 系统会返回这个区间中第二大的数的下标，在不超过20次询问找到隐藏数组中最大数位置的下标。

不超过20次询问显然是个二分，我们先整体询问 1,n 一次找到整个数组的第二大数位置，然后左右分别再询问一次，找到最大数是在第二大数的左边还是右边，确定方向后直接二分找最大数，二分决策是当前询问的区间所返回的第二大数是否还是第一次询问返回的下标，如果是就说明最大数就在询问区间中。

Building a Fence

Problem - C - Codeforces

给你 n 个长度为 k 的栅栏，摆放在长度为n的数轴上，根据栅栏编号从左到右依次摆放，数轴上每个点都有个初始高度 hi ，栅栏只能摆放在 hi 之上且下边界离 hi 不超过 k-1，相邻的栅栏要至少有长度为 1 的公共边，问能否满足这些摆放要求。

根据题意可以大致猜想出需要维护一个上下边界去check能否满足放栅栏的要求，对于第 i 个栅栏的上边界 u = min(ui-1 + k-1, 2*k-1+hi) 可以是最高的，但不能高过上个栅栏的上边界+k-1，对于第 i 个栅栏的下边界 d = max(di-1-k+1, hi+1) 可以只比 hi 高 1 ，但是不能低于上个栅栏的下边界-k+1。

最后一个栅栏的范围要特别判断。

Advertising Agency

Problem - E - Codeforces

有一个长度为 n 的数组，选其中 k 个，使其值最大，请问有几种选法。

首先肯定是选最大的 k 个，但为什么问几种选法？因为其中最后一种最小的数可能不止一个，那么直接组合数选就好了。

Longest Simple Cycle

Problem - C - Codeforces

给你 n 条链，每条链有 ai 个点，ai-1条边，把他们从上到下逐个连接起来，从第二条链开始，第 i 条链的第一个点连接第 i-1 条链的第 bi 个点，最后一个点连接第 i-1 条链的第 ci 个点，找到最大的环。

每一条链都被两条边连接起来，那么就从后往前看，如何选边才能最大环。

初始化最后一条链为主链，一直往前连接，如果当前两条边连接同一个点，这个环就被强制闭上了，需要重新找到一条主链，否则就说明当前有两个选择：闭环或者继续往前连接。遍历取所有最大值即可。

如果遇到当前主链比之前连接的点还要多，就把主链更新成当前的。

ai > bi或者 bi > ai 没有影响，因为找的是环，环转一圈两圈并不会发生什么。

Fence Painting

Problem - C - Codeforces

你现在要给 n 个栅栏涂色，原本每个栅栏的颜色是 ai，你希望它被涂成 bi ，有 m 个粉刷匠，每个粉刷匠会把一个栅栏涂色成 ci ，问怎么安排才能让所有栅栏都被涂色长期望颜色。

按顺序涂色问题显然要从后往前考虑，哪怕前面的涂色随便涂，只要后面涂正确就行。

对于第 i 个粉刷匠的颜色 ci ，如果刚好有一个地方需要被涂色成 ci 就把他分配到这个位置，如果没有就再找有没有原本就是 ci 颜色的栅栏，把这个地方分配给他，这样的话涂了没有影响，如果还没有的话可以把他分配给后面已经确定的会被涂色的方块，这样哪怕现在涂的是错误的颜色，后面也会修正。

如果当前 ci 既不需要被涂，后面也没有已经确定的需要涂色的，就说明当前不可能涂色成功。

Basic Diplomacy

Problem - C - Codeforces

你有 n 个朋友，需要一起度假 m 天，每天你需要选择一位朋友一起游戏，每位朋友的总游戏次数不能超过 m/2 上取整次。怎样安排每天的朋友？

贪心的考虑，每天可以选择的朋友数不同，先要尽可能的去优先决定那些选择少的天数， 在这一天的选择中，又需要优先选择见面次数较少的朋友，如果这次不选他，那么就没有下次选他的机会了。
