Training Session

Problem - D - Codeforces

n 个问题，每个问题都有两个参数a b，保证没有两个问题的a b 是相同的，从这 n 个问题中选出三个问题，要求满足：三个问题的参数 a 是互不相同的，或三个问题的参数 b 是互不相同的。求所有选择问题的方案数。

数数问题，不考虑条件，直接选的话，所有的方案数是 C(n,3) 。在所有的方案中包含：只满足条件1，只满足条件2，同时满足条件1和2，同时不满足条件1和2，题目要求出前三种，所以可以只计算第四种情况，用所有方案数减去就是最终求的。

同时不满足条件1和2说明所选择的三个问题中，存在两个问题的参数 a 是相同的，且存在两个问题的参数 b 是相同的，因为已经保证了没有两个问题的参数 a b 是相同的，所以选择出来的方案一定是 [(a,b) ,(a,x), (y,b)] 形式的，枚举每一对 a b，用总方案累减就行了。

The Number of Imposters

Problem - 1594D - Codeforces

n 个人在玩狼人杀，好人说的话一定是真的，坏人说的话一定是假的，他们说了 m 句话，第 i 句有两个参数 x y ，和一个字符串，表示 x 说 y 是 好人/坏人。在这局游戏中最坏情况下会有多少坏人？如果矛盾输出 -1。

第一感觉就是要用并查集维护他们的关系，仔细思考后发现与食物链类似，可以用扩展域去解决。

设 [1~n] 为玩家 i 是好人，[n+1~n+n] 为玩家 i 是坏人。

根据游戏规则：

x 说 y 是好人 ，如果 x 是好人，说明 y 也是好人，如果 x 是坏人，说明 y 也是坏人。

x 说 y 是坏人 ，如果 x 是好人，说明 x 是坏人，如果 x 是坏人，说明 y 是好人。

根据规则将扩展域连接好，然后对于每一个 i 和 i+n 选择坏人多的累加就行了。

Bakry and Partitioning

Problem - C - Codeforces

给你一个由 n 个点组成的树结构，每个点都有一个值 ai，你可以进行删除这棵树不超过 k-1 条边且不少于 1 条边的操作，求是否可以将这棵树分割成若干课子树，使得每科子树上每个节点的异或和相等。

设整棵树每个节点的异或和为 sum。

根据异或的性质可知：如果这棵树可以分割成偶数个符合条件的子树，那么这些子树的异或和再异或等于 0，也就是说 sum = 0 。如果这棵树可以分割成奇数个符合条件的子树，那么它一定可以正好分割成三颗子树，且每颗子树的异或和为 sum。

根据树结构的性质可知：任意删除一条边就可以分割出来一颗子树。

综上，符合题解的情况为：

sum = 0，k为任何值，

sum = x，k>=3，dfs这棵树，计数异或和为 x 的子树的数目，且把这颗子树切掉，最终符合条件的子树数目 >=3。

Omkar and Determination

Problem - C - Codeforces

在一个 n * m 大小的平台上有两种方块，”.” 可以移动，”#” 不可以移动，可以移动的方块只能向左或向上移动。一个方块可以移动出这个棋盘就标记为E，否则标记为X。问题是：是否可以根据这个平台上每个方块可否移动来判断出他们原本的方块种类？

标记为E的方块一定是可移动方块，标记为X的方块则需要分情况讨论。

​ XX ## ##
对于 XX 这样的标记，它可以是 # . ，也可以是 ## ，也就是说，如果一个方块的左边和上方都是不可移动方块，那么它就是一个不可判断的。

Portal

Problem - A - Codeforces

给你一个 n * m 的01矩阵，你可以把任意位置的数字翻转，找到一个长大于等于5宽大于等于4的子矩阵，使得矩阵的四条边除了四个角以外其他的位置都是 1 ，矩阵内部都是 0，且操作数尽可能的少。

很容易可以想到二位前缀和维护矩阵01的数量，然后四重循环枚举上下左右边界。考虑优化最后一维， 上下左边界确定了，右边界的枚举过程中，矩形内部的翻转次数一定是随着右边界的扩大而递增的，当矩形内部的翻转次数大于了右边界的长度，就不需要再继续扩展右边界了，也就是说在向右扩展右边界的过程中，如果当前划出来的区域翻转次数大于之前划出来的区域翻转次数就跳出循环。

矩阵只有01两种数字，右边界上1多了，边界翻转次数较少，但是下次会贡献给内部很多的翻转次数。右边界上0多了，边界翻转次数就多。所以向右扩展不会进行多少次，总的复杂度是n三次方的。
